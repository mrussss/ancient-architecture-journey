# Codex 开发 Prompt：重构为赵州桥叙事导览关

你现在在一个已经能运行的 C++17 + SDL2 + SDL_image + CMake 横版游戏项目中工作。

当前项目已经具备：

* SDL2 窗口与主循环；
* SDL_image PNG 素材加载；
* TextureManager 纹理管理；
* 玩家移动、跳跃、重力、平台碰撞；
* 金币收集；
* 终点通关；
* R 重启；
* ESC 退出；
* 赵州桥主题代码背景；
* assets/images 下已有最小像素素材包。

现在需要根据新的设计目标，对第一关进行重构。

---

# 一、重构目标

把当前“马里奥式跑酷平台跳跃 Demo”重构为：

> 中国古风赵州桥叙事导览关

本关不再以跑酷、跳平台、收集金币为核心，而是以：

* 看桥；
* 遇到 NPC；
* NPC 介绍赵州桥；
* 玩家走上桥；
* 桥中央触发涨水泄洪演示动画；
* 演示结束后玩家过桥进入下一关；

为主要流程。

---

# 二、核心设计原则

## 1. 去掉跑酷感

本轮需要弱化或移除跑酷元素：

* 不再需要金币收集作为核心目标；
* 不再需要分散的跳跃平台；
* 不再需要玩家跳上跳下；
* 不再需要复杂平台挑战；
* 不再强调分数。

本关目标是：

> 横版行走 + 触发对话 + 看桥动画 + 进入下一关。

可以保留最基础的碰撞和地面行走，但不要再设计跳跃跑酷路线。

---

## 2. 赵州桥是画面主体

赵州桥应该占据画面中心，是本关最重要的视觉对象。

要求：

* 桥体要比当前更精细；
* 桥面、桥栏杆、主拱、小拱都要清晰；
* 视觉上要有细小像素块质感；
* 石块、砖缝、栏杆等可以用 PNG 素材和代码辅助绘制共同完成；
* 玩家应当从桥左侧走到桥上，再走到桥中央。

---

## 3. 全局保持像素风

所有画面元素都应保持像素风：

* 主角；
* NPC；
* 桥；
* 水流；
* 对话框；
* UI；
* 动画效果。

必须设置：

```cpp
SDL_SetHint(SDL_HINT_RENDER_SCALE_QUALITY, "0");
```

不要使用模糊缩放。
如果需要缩放素材，尽量使用整数倍缩放或保持清晰的像素边缘。

# 四、第一关流程设计

请把第一关重构为以下流程：

## 阶段 1：玩家出生在桥左侧

玩家初始位置应在赵州桥左侧地面或桥头附近。

要求：

* 玩家出生点在画面左边；
* 不能一出生就在桥中央；
* 左侧应有 NPC 李春；
* 玩家可以向右行走；
* 当前阶段不需要跳跃挑战。

---

## 阶段 2：遇到 NPC 触发对话

当玩家靠近 NPC 李春时，进入对话状态。

要求：

* NPC 使用 `assets/images/npc/lichun_idle.png`；
* 对话时可以显示 `assets/images/npc/lichun_portrait.png`；
* 显示 galgame 风格对话框；
* 使用 `assets/images/ui/dialog_box.png`；
* 使用 `assets/images/ui/name_box.png`；
* 使用 `assets/images/ui/next_arrow.png`；
* 对话框在屏幕底部；
* 显示说话人名字；
* 按 Space / Enter / 鼠标左键进入下一句；
* 对话期间暂停玩家移动；
* 对话结束后恢复玩家控制。

---

## 阶段 3：NPC 介绍赵州桥

请加入一组分段对话，内容不需要太长，但要有文化介绍。

建议对话内容：

1. 李春：
   “你来了。眼前这座桥，便是赵州桥。”

2. 李春：
   “它建于隋代，由工匠李春主持修建，是中国古代石拱桥的杰出代表。”

3. 李春：
   “赵州桥最特别的地方，不只是桥身坚固，更在于它独特的敞肩拱结构。”

4. 李春：
   “你可以走到桥中央，亲眼看看水流经过桥洞时的变化。”

5. 小研：
   “原来古代桥梁不仅要能通行，还要懂得顺应水势。”

要求：

* 文字可以先写死在代码中；
* 后续再抽成文本文件也可以，但本轮不是必须；
* 对话系统要结构清晰，方便后续添加第二关 NPC 对话。

---

## 阶段 4：玩家走上赵州桥

对话结束后，玩家继续向右走。

要求：

* 玩家走上桥面；
* 桥面应是主要行走区域；
* 不要再出现明显的跳跃平台路线；
* 可以把原来的分散平台隐藏、删除或重构成桥面碰撞区域；
* 玩家应主要沿着桥面水平移动；
* 如果桥面素材是弧形，碰撞可以先用简单矩形或分段矩形近似，不要求真实弧形碰撞。

---

## 阶段 5：桥中央触发动画

当玩家走到赵州桥中央区域时，触发演示动画。

触发条件建议：

```text
玩家 x 坐标进入桥中央 trigger 区域。
```

触发后进入 Cutscene 状态：

* 暂停玩家输入；
* 玩家可切换为 `player_observe.png`；
* 镜头/画面保持在桥中央；
* 播放涨水与泄洪演示；
* 显示说明文字。

---

## 阶段 6：涨水泄洪演示动画

动画目标：

> 表现涨水时，水流可以通过主拱和两侧小拱流出，从而减轻桥体压力。

注意：不要做真实流体模拟，只做视觉演示动画。

建议动画分为 4 个阶段：

### 1. 水位上升

* 桥下河水水位逐渐上升；
* 水面颜色可以变深；
* 水波纹变密；
* 持续约 1.5 秒。

### 2. 桥洞高亮

* 用浅金色或浅蓝色高亮主拱和两侧小拱；
* 可以画半透明线条或闪烁描边；
* 表示“水流通过桥洞”。

### 3. 水流通过桥洞

* 在主拱和小拱下方绘制水平移动的水波纹；
* 水波纹从左向右或从中间向两侧流动；
* 使用 `water_wave_strip.png` 或代码绘制像素短线；
* 保持像素风，不要使用平滑曲线。

### 4. 显示说明文字

显示一段简短说明：

```text
“涨水时，主拱与两侧小拱共同分流水流，减轻桥体压力，也让桥身更加稳固。”
```

动画结束后，恢复玩家控制。

---

## 阶段 7：过桥进入下一关

演示动画结束后，玩家继续向右走。

当玩家到达桥右侧出口区域时，进入下一关或显示过关提示。

当前如果还没有第二关，可以先实现：

* 显示 “第一关完成：赵州桥”；
* 或显示 “即将进入下一关：徽派民居”；
* 或进入一个占位 NextLevel 状态。

不要直接退出程序。

# 五、建议新增游戏状态

请将游戏状态从原来的 Playing / Win / GameOver，重构或扩展为：

```cpp
enum class GameState {
    Playing,
    Dialog,
    Cutscene,
    LevelComplete
};
```

如果原本已有状态机，请在原基础上扩展，不要粗暴删除所有旧代码。

状态说明：

* `Playing`：玩家可控制；
* `Dialog`：对话中，玩家不可移动；
* `Cutscene`：桥中央演示动画中，玩家不可移动；
* `LevelComplete`：第一关完成或准备进入下一关。
